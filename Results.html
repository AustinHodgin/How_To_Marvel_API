How To Make API Calls

You are probaby wondering what this API does. The Marvel API will allow you to pull information from the marval servers about all kinds of diffrent charaters, story archs, events, comic covers, and a bunch more information. To get his information you will need to make a call to

http://gateway.marvel.com/

If you would like to see what to add to the end of http://gateway.marvel.com/ you will need to go here. I will just show you a few of them. Below is an example of a call you would make if you are looking for a charater:

http://gateway.marvel.com:80/v1/public/characters?name=NAMEOFHERO&apikey=YOURAPIKEY

Lets break this down a little bit. The first part:
http://gateway.marvel.com/
Is the base url for the server. This is where the information is stored.

/v1/public/characters
Is telling the server that you are looking for a characther name.

?name=NAMEOFHERO
Is the name of the hero that you are looking for

apikey=YOURAPIKEY
This is your API Key that you got from the last


If you would like to see how more of these calls are made you are can test all the diffrent calls here in the documntation.

http://gateway.marvel.com:80/v1/public/characters?name=Deadpool&apikey=MYAPIKEY

You're probably thinking that is a lot of information. What does it all mean? well lets break it down so that when you are parsing this information in your javascript you will know what to look for.
"code": 200, "status": "Ok", "copyright": "© 2016 MARVEL", "attributionText": "Data provided by Marvel. © 2016 MARVEL", "attributionHTML": "<a href=\"http://marvel.com\">Data provided by Marvel. © 2016 MARVEL</a>",
code is the code that is given to the charater.
status is if the request from the server was sucessful.
copyright is who is the copyright holder of the charater. This will almost always be MARVEL.
attribution text is where the data is provided and who owns the copyright.
